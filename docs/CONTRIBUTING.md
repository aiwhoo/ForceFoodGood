## Contributing Guidelines

Thank you for contributing to ForceFoodGood!

### Workflow
1. Clone the repository
2. Create a new branch from main (e.g., feature-add-cart)
3. Make your changes
4. Pull the latest changes from main before committing
5Commit your changes with clear messages
5. Push your branch
6. Create a Pull Request

### Coding Standards
- Use clear and meaningful variable names
- Keep functions small and readable
- Comment important parts of your code

### Object-Oriented Programming (OOP)
- Follow OOP principles (Encapsulation, Abstraction, Modularity)
- Keep classes focused on a single responsibility
- Avoid duplicate code by reusing classes and methods

### Issues
- Check existing issues before creating a new one
- Assign yourself before working on an issue

### Communication
- Review existing code before making changes
- Communicate with teammates if your work may overlap
- Ask questions early if something is unclear
- Update progress in issues or pull requests

### Conflict Resolution
- Always pull the latest changes before pushing
- Resolve merge conflicts carefully before submitting PR
- Clearly explain conflicts and how you resolved them
- Ensure your branch merges cleanly into main
